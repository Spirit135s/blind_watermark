# 图像频域盲水印原理讲解

## 1. 项目算法定位

本项目实现的是 **DWT-DCT-SVD 频域盲水印**。它把水印信息隐藏到图像的频域系数中，生成肉眼难以察觉差异的含水印图像；提取时不需要原始图像，只需要含水印图像、密钥以及水印长度或形状。

这里的“盲水印”不是完全没有先验信息，而是指提取阶段不依赖原始未加水印图像。实际提取仍需要：

- `password_img`：决定图像频域块内系数置乱顺序。
- `password_wm`：决定水印 bit 的置乱顺序。
- `wm_shape`：水印长度或图片水印的宽高。
- 与嵌入时一致的算法参数，尤其是 `d1/d2`。

## 2. 嵌入流程

整体流程可以概括为：

```text
原图
  -> BGR 转 YUV
  -> 对 Y/U/V 三个通道分别做 DWT
  -> 取低频子带 ca
  -> 将 ca 切成 4x4 小块
  -> 每个小块做 DCT
  -> 按 password_img 打乱 DCT 系数
  -> 对打乱后的 4x4 系数矩阵做 SVD
  -> 修改奇异值 s[0] 和 s[1] 表示水印 bit
  -> 逆 SVD、逆置乱、逆 DCT
  -> 替换低频子带并做逆 DWT
  -> YUV 转 BGR，得到含水印图
```

源码中的核心位置是 `blind_watermark/bwm_core.py`：

- `read_img_arr()`：完成图像预处理、YUV 转换、DWT 分解和低频分块。
- `block_add_wm_slow()`：对单个 4x4 低频块嵌入 1 bit 水印。
- `embed()`：遍历三个通道和所有低频块，完成整张图的嵌入和逆变换。

水印可以来自三种输入：

- 图片水印：先转灰度，再按阈值变成 bit 序列。
- 字符串水印：先按 UTF-8 编码，再转二进制 bit 序列。
- bit 序列：直接作为水印 bit 使用。

这些转换由 `blind_watermark/blind_watermark.py` 中的 `WaterMark.read_wm()` 完成。

## 3. DWT、DCT 与 SVD 的作用

### DWT：选择较稳定的频域区域

DWT 即离散小波变换。对图像做 Haar 小波分解后，会得到低频部分 `ca` 和高频细节部分 `hvd`。

项目选择在低频 `ca` 中嵌入水印，原因是：

- 低频包含图像主要结构信息。
- JPEG 压缩、缩放、轻微噪声等攻击通常更容易破坏高频细节。
- 在低频中做适量修改，水印更容易在攻击后保留下来。

代价是低频改动过大会影响视觉质量，所以需要控制嵌入强度。

### DCT：在局部频域块中嵌入信息

DCT 即离散余弦变换。项目把 DWT 低频子带切成 `4x4` 小块，再对每个小块做 DCT。

这样做的意义是：

- 将局部图像块从空间域转到频域。
- 水印分散到大量小块中，而不是集中在某个区域。
- 即使图像局部被裁剪、遮挡或加噪，也仍可能从其他块中恢复水印。

### SVD：用奇异值承载 bit

SVD 即奇异值分解。对每个 DCT 后的小块做：

```text
A = U * S * V
```

项目主要修改奇异值 `s[0]`，并可同时修改 `s[1]`。奇异值具有较好的稳定性，图像受到轻微扰动后仍能保持一定可识别性，因此适合承载水印。

项目用奇异值所在的模区间表示 0/1：

```text
s[0] = (s[0] // d1 + 1/4 + 1/2 * wm_bit) * d1
s[1] = (s[1] // d2 + 1/4 + 1/2 * wm_bit) * d2
```

当 `wm_bit = 0` 时，奇异值被放到周期内靠前的位置；当 `wm_bit = 1` 时，奇异值被放到周期内靠后的位置。提取时只需要判断奇异值落在周期的前半区还是后半区。

## 4. 提取流程

提取流程和嵌入流程前半段一致，但不需要原始图像：

```text
含水印图
  -> BGR 转 YUV
  -> DWT
  -> 低频子带分块
  -> DCT
  -> 按 password_img 恢复同样的系数置乱顺序
  -> SVD
  -> 判断奇异值模区间得到每块的 bit 估计
  -> 对重复嵌入结果求平均
  -> 对字符串/bit 水印使用一维 k-means 判决 0/1
  -> 按 password_wm 反置乱
  -> 恢复成图片、字符串或 bit 序列
```

核心源码位置：

- `extract_raw()`：从每个通道、每个块中提取 1 个 bit 估计。
- `extract_avg()`：同一个水印 bit 会被循环嵌入多次，这里对重复观测求平均。
- `extract_with_kmeans()`：对字符串和 bit 水印，用一维 k-means 自动寻找 0/1 阈值。
- `WaterMark.extract_decrypt()`：用 `password_wm` 反向恢复水印 bit 顺序。

## 5. 关键参数说明

- `password_img`：图像侧密钥，用来生成每个块内 DCT 系数的随机置乱顺序。嵌入和提取必须一致。
- `password_wm`：水印侧密钥，用来打乱水印 bit 顺序。提取后用同样的随机顺序反置乱。
- `block_shape`：低频子带分块大小，当前核心实现固定使用 `4x4`。
- `d1/d2`：嵌入强度参数。越大，水印越容易抵抗攻击，但图像失真越明显。
- `wm_shape`：提取时必须提供。字符串和 bit 水印一般是长度，图片水印是二维形状。
- `mode`：并行模式，可选择普通串行、多线程或多进程，不改变算法本身。

## 6. 不可见性与鲁棒性

不可见性来自两点：

1. 水印被写入频域，而不是直接修改明显的空间域像素。
2. 每个块只调整少量奇异值，且修改幅度由 `d1/d2` 控制。

鲁棒性来自三点：

1. 水印写在 DWT 低频部分，低频比高频更抗压缩和缩放。
2. SVD 奇异值对轻微扰动相对稳定。
3. 水印 bit 在多个通道和多个块中重复嵌入，提取时通过平均降低局部攻击的影响。

因此，强度参数存在典型权衡：

- `d1/d2` 较小：图像质量更好，PSNR 更高，但抗攻击能力较弱。
- `d1/d2` 较大：攻击后更容易提取，但含水印图和原图差异更明显。

后续大作业实验应围绕这个权衡展开，统计不同强度下的 PSNR、MSE 和提取成功率。

## 7. 与大作业要求的对应关系

| 作业要求 | 项目对应实现 |
| --- | --- |
| 在图像频域嵌入不可见数字水印 | 使用 DWT-DCT-SVD 在频域低频块中嵌入 bit |
| 通过逆变换获得含水印图像 | 修改奇异值后执行逆 SVD、逆 DCT、逆 DWT |
| 验证水印不可见性 | 可通过原图/含水印图对比、差异图、MSE、PSNR 量化 |
| 从含水印图像中提取原始水印 | `WaterMark.extract()` 支持图片、字符串和 bit 水印提取 |
| 分析嵌入强度对图像质量影响 | 可扫描 `d1/d2`，记录 PSNR/MSE 和提取正确率 |
| 测试抗打击能力 | `att.py` 已提供缩放、裁剪、旋转、遮挡、椒盐、亮度等攻击函数 |
| 设计盲水印算法 | 该项目提取时不需要原始图像，符合盲水印要求 |

## 8. 后续实验建议

为了形成完整的大作业结果，建议新增实验脚本完成：

1. 固定原图和水印，扫描多组 `d1/d2`。
2. 对每组强度生成含水印图、差异热力图、PSNR/MSE。
3. 对含水印图做 JPEG 压缩、缩放、加噪等攻击。
4. 攻击后提取水印，记录提取成功率或 bit error rate。
5. 输出表格和曲线，用于报告分析。

这样可以把当前项目从“可用的盲水印库”整理成“满足课程要求的完整实验系统”。
